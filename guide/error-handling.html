error handling

Jade has no exceptions. errors are values.

why no exceptions?

exceptions are invisible. when you call a function that throws, nothing in its signature tells you that. you have to read the docs — or find out the hard way at runtime.

// Java — can this throw? who knows
String result = someService.fetchData(url);

in Jade, if a function can fail, its return type says so:

def fetchData(url str) Result<str> { ... }

you can't ignore it. the compiler makes you handle it.

Result<T>

Result<T> is a sealed class with two variants:

sealed class Result<T> {
    data class Ok(val value T)   // success
    data class Err(val msg str)  // failure
}

functions that can fail return Result<T>:

def divide(a f64, b f64) Result<f64> {
    if b == 0.0 {
        return Result.Err("division by zero");
    }
    return Result.Ok(a / b);
}

handling results

use when to handle both cases:

val result = divide(10.0, 2.0);

when result {
    is Ok(v)  -> io.println("result: #{v}");
    is Err(e) -> io.println("error: #{e}");
}

chaining results

def parseInt(s str) Result<i32> { ... }
def double(n i32) Result<i32> = Result.Ok(n * 2);

val result = parseInt("21")
    .map(def(n i32) i32 = n * 2)
    .unwrapOr(0);

io.println("#{result}");  // 42

useful Result methods

result.unwrap()          // get value or crash
result.unwrapOr(default) // get value or use default
result.map(fn)           // transform Ok value
result.mapErr(fn)        // transform Err message
result.isOk()            // bool
result.isErr()           // bool

Option<T>

for values that might not exist (like nullable types):

sealed class Option<T> {
    data class Some(val value T)
    data class None()
}

def findUser(id u32) Option<User> {
    // returns Some(user) if found, None() if not
}

when findUser(42) {
    is Some(user) -> io.println("found: #{user.name}");
    is None()     -> io.println("user not found");
}

alternatively use nullable types with ?:

def findUser(id u32) User? {
    // returns null if not found
}

val user = findUser(42);
if user != null {
    io.println("found: #{user.name}");
}

propagating errors

in deeply nested code, use early returns:

def processFile(path str) Result<str> {
    val exists = fs.exists(path);
    if !exists {
        return Result.Err("file not found: #{path}");
    }

    val content = fs.read(path);
    when content {
        is Err(e) -> return Result.Err("read failed: #{e}");
        is Ok(v)  -> return Result.Ok(v.toUpperCase());
    }
}

panic

for truly unrecoverable errors (programmer mistakes, not user errors):

def getIndex(arr []i32, i i32) i32 {
    if i < 0 || i >= arr.len() {
        panic("index #{i} out of bounds for array of length #{arr.len()}");
    }
    return arr[i];
}

panic crashes the program with a message and stack trace. use it sparingly — only when the program is in an invalid state that can't be recovered from.
