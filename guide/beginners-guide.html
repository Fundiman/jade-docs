learn Jade — a beginner's guide

Jade is a write once, run anywhere programming language.

Ruby's elegance + Go's simplicity + Kotlin's type safety.

Curly braces. Semicolons. No excuses.

before you start

you need:

  • jscc — the Jade compiler
  • jvk — the Jade runtime
  • VS Code + the Jade extension (for syntax highlighting)

build them from github.com/Fundiman/JadeLang or use jpkg.

chapter 1 — hello world

every Jade program starts with a package declaration and an entry point.

package com.yourname.hello;

import jade.stdlib.io;

@entry
def main(args []str) i32 {
    io.println("hello, world!");
    return 0;
}

save it as main.jsc, then:

jscc main.jsc out.joc
jvk out.joc

output:

hello, world!

what's happening:

  • package — declares where this code lives, like an address
  • import — brings in the standard library
  • @entry — tells jvk this is where execution starts
  • def main — defines a function called main
  • args []str — main receives a list of strings (command line arguments)
  • i32 — main returns a 32-bit integer (0 = success)
  • io.println — prints a line to the console
  • return 0 — exit with success

chapter 2 — variables

Jade has two kinds of variables:

val name = "Jade";    // immutable — cannot be changed
var count = 0;        // mutable — can be changed

val is like a constant. once set, it stays that way forever. var can be updated whenever you need.

val x = 10;
x = 20;        // ERROR — val is immutable!

var y = 10;
y = 20;        // fine

types

Jade can figure out the type automatically (type inference):

val name = "Jade";      // str
val age  = 12;          // i32
val pi   = 3.14159;     // f64
val flag = true;        // bool

or you can write the type yourself:

val name str  = "Jade";
val age  i32  = 12;
val pi   f64  = 3.14159;
val flag bool = true;

number types

typewhat it is
i32integer (-2 billion to 2 billion)
i64big integer
u32positive integer only
f64decimal number (double)
f32smaller decimal number

string interpolation

put variables inside strings with #{}:

val name    = "Jade";
val version = 1;

io.println("hello from #{name} v#{version}!");
// prints: hello from Jade v1!

io.println("2 + 2 = #{2 + 2}");
// prints: 2 + 2 = 4

chapter 3 — functions

def add(a i32, b i32) i32 {
    return a + b;
}
  • def — declares a function
  • add — the function name (camelCase)
  • (a i32, b i32) — parameters: name then type
  • i32 after ) — return type
  • return — sends back a value

single expression functions

if your function is just one expression, use =:

def add(a i32, b i32) i32 = a + b;

def greet(name str) str = "hello #{name}!";

default arguments

def greet(name str = "world") str = "hello #{name}!";

greet();           // "hello world!"
greet("Jade");     // "hello Jade!"

multiple return values

def minMax(arr []i32) (i32, i32) {
    return (0, 100);   // (min, max)
}

val (lo, hi) = minMax(numbers);

async functions

async def fetchData(url str) Result<str> {
    val response = http.get(url).await();
    return response;
}

chapter 4 — control flow

if / else

val x = 42;

if x > 100 {
    io.println("big");
} else if x > 50 {
    io.println("medium");
} else {
    io.println("small");
}

for loop

// loop over a range
for i in 0..10 {
    io.println("#{i}");
}

// loop over a collection
val names = ["Alice", "Bob", "Charlie"];
for name in names {
    io.println("hello #{name}!");
}

while loop

var count = 0;
while count < 5 {
    io.println("count: #{count}");
    count += 1;
}

when (pattern matching)

when is like a supercharged switch statement:

val x = 42;

when x {
    is 1  -> io.println("one");
    is 42 -> io.println("the answer!");
    is _  -> io.println("something else");
}

it's most powerful with sealed classes (see chapter 6).

chapter 5 — collections

arrays

val numbers = [1, 2, 3, 4, 5];
val names   = ["Alice", "Bob", "Charlie"];

io.println("#{numbers[0]}");    // 1
io.println("#{names.len()}");   // 3

list (dynamic array)

import jade.stdlib.collections;

var list = List<i32>.new();
list.push(1);
list.push(2);
list.push(3);

io.println("#{list.len()}");    // 3
io.println("#{list.get(1)}");   // 2

map (dictionary)

var scores = Map<str, i32>.new();
scores.set("Alice", 95);
scores.set("Bob",   87);

io.println("#{scores.get("Alice")}");  // 95

set (unique values)

var unique = Set<str>.new();
unique.add("apple");
unique.add("banana");
unique.add("apple");   // duplicate — ignored

io.println("#{unique.len()}");  // 2

chapter 6 — classes

data class

a data class is a simple container for values:

data class Point(val x f64, val y f64);

val p = Point(1.0, 2.0);
io.println("#{p.x}, #{p.y}");  // 1, 2

regular class

class Counter {
    var count i32

    def increment() {
        count += 1;
    }

    def value() i32 = count;
}

val c = Counter.new();
c.increment();
c.increment();
io.println("#{c.value()}");  // 2

generic class

class Box<T> {
    val item T

    def get() T = item;
}

val box = Box<str>.new("hello");
io.println(box.get());  // hello

sealed class (enums with data)

sealed classes are one of Jade's most powerful features:

sealed class Shape {
    data class Circle(val radius f64)
    data class Rectangle(val w f64, val h f64)
    data class Triangle(val base f64, val height f64)
}

def area(s Shape) f64 {
    return when s {
        is Circle(r)        -> 3.14159 * r * r;
        is Rectangle(w, h)  -> w * h;
        is Triangle(b, h)   -> 0.5 * b * h;
    };
}

val c = Shape.Circle(5.0);
io.println("area: #{area(c)}");  // area: 78.5397

traits (interfaces)

trait Printable {
    def print() void;
}

class Dog {
    val name str

    def print() void {
        io.println("dog: #{name}");
    }
}

chapter 7 — error handling

Jade uses Result<T> instead of exceptions:

sealed class Result<T> {
    data class Ok(val value T)
    data class Err(val msg str)
}

def divide(a f64, b f64) Result<f64> {
    if b == 0.0 {
        return Result.Err("division by zero");
    }
    return Result.Ok(a / b);
}

val result = divide(10.0, 2.0);

when result {
    is Ok(v)  -> io.println("result: #{v}");
    is Err(e) -> io.println("error: #{e}");
}

no try/catch. no exceptions flying around unpredictably. errors are just values you handle explicitly.

chapter 8 — type aliases

give complex types a readable name:

type UserId   = u32;
type Handler  = def(Request) Response;
type Matrix   = [][]f64;
type MaybeInt = i32?;

now instead of writing def(Request) Response everywhere you just write Handler.

chapter 9 — decorators

decorators modify functions at runtime:

// built-in decorators
@entry          // marks the program entry point
@Suppress("JD601")  // suppress a compiler warning

// web framework decorators
val app = WebApp.new();

@app.route("/")
def index(req Request) Response {
    return Response.ok("hello!");
}

@app.route("/users", method = "POST")
def createUser(req Request) Response {
    val name = req.body.get("name");
    return Response.created("created #{name}");
}

chapter 10 — packages and imports

// declare your package
package com.yourname.myapp;

// import standard library modules
import jade.stdlib.io;
import jade.stdlib.math;
import jade.stdlib.fs;
import jade.stdlib.os;
import jade.stdlib.collections;

// import specific things from a package
import jade.stdlib.collections.{ List, Map, Set };

jpkg — the package manager

jpkg init                        # create jade.toml
jpkg add com.someone.coolpackage # add a dependency
jpkg install                     # install all deps
jpkg build                       # compile your project
jpkg run                         # build and run

your jade.toml:

[package]
name    = "com.yourname.myapp"
version = "1.0.0"
license = "LGPL-3.0"

[dependencies]
"jade.stdlib" = "0.1.0-alpha"

[build]
entry  = "src/main.jsc"
output = "out/main.joc"

chapter 11 — the .joc format

when you compile a .jsc file you get a .joc file. .joc is a fat binary — it contains:

  • native machine code for your architecture
  • LLVM bitcode fallback (runs on any architecture via JIT)
jscc main.jsc                          # host + bytecode (default)
jscc main.jsc --target=aarch64-linux   # add ARM64 slice
jscc main.jsc --bytecode-only          # portable only
jscc main.jsc --no-bytecode            # native only, smaller
jvk main.joc                 # run (picks best slice automatically)
jvk main.joc --jit-only      # force JIT
jvk main.joc --info          # show what's inside the .joc
