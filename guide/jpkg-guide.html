jpkg — the Jade package manager

installation

jpkg is a Python script — no installation needed:

python3 path/to/JadeLang/jpkg/jpkg.py --version
# jpkg 0.1.0-alpha

add an alias for convenience:

alias jpkg="python3 /path/to/JadeLang/jpkg/jpkg.py"

creating a project

mkdir myapp && cd myapp
jpkg init

creates:

myapp/
  jade.toml       ← project configuration
  src/
    main.jsc      ← hello world entry point
  out/            ← compiled output goes here

jade.toml

[package]
name        = "com.yourname.myapp"
version     = "1.0.0"
license     = "LGPL-3.0"
description = "my Jade app"
authors     = ["yourname"]

[dependencies]
"jade.stdlib"          = "0.1.0-alpha"
"com.someone.coolpkg" = "^1.2.0"

[build]
entry   = "src/main.jsc"
output  = "out/main.joc"
targets = ["aarch64-linux"]   # extra arch slices

commands

jpkg add

add a dependency:

jpkg add jade.stdlib.math
jpkg add com.someone.package@2.0.0
jpkg add jade.stdlib.http@latest

updates jade.toml and shows the resolved version.

jpkg install

download and install all dependencies:

jpkg install

packages are cached in ~/.jade/registry/ and symlinked into jade_modules/.

jpkg remove

jpkg remove jade.stdlib.math

removes from jade.toml and jade_modules/.

jpkg update

update all dependencies to the latest compatible versions:

jpkg update

respects semver constraints — ^1.0.0 will update to 1.9.9 but not 2.0.0.

jpkg list

show installed packages and their status:

jpkg list
package: com.yourname.myapp v1.0.0

dependencies:
  jade.stdlib            0.1.0-alpha    ✓
  jade.stdlib.math       ^1.0.0         ✓
  com.someone.package    ^2.0.0         ✗ not installed

jpkg build

compile your project:

jpkg build

calls jscc with the flags from jade.toml. equivalent to:

jscc src/main.jsc out/main.joc --target=aarch64-linux

jpkg run

build and run:

jpkg run
jpkg run -- --my-app-flag value   # pass args to your program

jpkg search

search the registry:

jpkg search http
jpkg search json

jpkg info

get details about a package:

jpkg info jade.stdlib.math

publishing a package

prerequisites

  • a GitHub account
  • git installed
  • gh CLI installed and authenticated (gh auth login)

publish

jpkg publish

this will:

  1. build your .joc if it doesn't exist
  2. fork Fundiman/jade-pkgs to your GitHub account
  3. create a branch pkg/your.package.name/version
  4. add your jade.toml and out.joc
  5. open a pull request automatically

a maintainer will review and merge. once merged, anyone can jpkg add your.package.name.

semver constraints

constraintmeaning
1.2.3exactly 1.2.3
^1.2.3>=1.2.3 and <2.0.0 (same major)
~1.2.3>=1.2.3 and <1.3.0 (same minor)
>=1.2.0at least 1.2.0
latestwhatever the registry says is latest
*any version

local packages

reference a local directory as a dependency:

[dependencies]
"com.mine.utils" = { path = "../utils" }

useful during development before publishing.
